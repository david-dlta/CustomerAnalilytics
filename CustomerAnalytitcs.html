Objetivo

El objetivo del proyecto es analizar datos transaccionales de un e-commerce y segmentar los clientes.

Se va a dividir en cinco partes:

  1. Importacin de datos, anlisis de calidad de los datos y anlisis descriptivo.
  2. Preparacin de los datos.
  3. Segmentacin de clientes con la recencia y la frecuencia.
  4. Segmentacin de clientes con el gasto total y frecuencia de compra.
  5. Dendograma de tipologa de productos por el gasto de cada cliente.

Opcionalmente, se va a aplicar un mtodo de segmentacin analtica no supervisado mediante un modelo kmeans. Adicionalmente se va a lleavar a cabo un Anlisis de Componentes Principales, PCA.

2. Preparacin de los datos

En este bloque vamos a realizar todas las transformaciones de datos necesarias para posteriormente trabajar con ellos. Tenemos que acomodar los datos a las necesaidades que vamos a tener.

Lo primero va a ser transformar las fechas a formato Date, actualmente se encuentran como strings.

df_RAW <- df_RAW %>%
  mutate(
    orderdate = as.Date(orderdate, '%d/%m/%Y')
  )

Continuamos preparndo los campos que vamos a necesitar para calcular las variables que conforman el RFM.

df <- df_RAW %>%
  group_by(clientId) %>%
  summarise(
    firstOrderDate = min(orderdate),
    lastOrderDate = max(orderdate), 
    n_Orders = n_distinct(orderId),
    n_Products = n_distinct(product),
    totalSpent = sum(Quantity*Price)
  )
## `summarise()` ungrouping output (override with `.groups` argument)
date <- max(df$lastOrderDate) #Nos situamos en la fecha en la que vamos a realizar el anlisis

El siguiente paso es calcular la RFM. Para ello deberemos calcular la Recencia, la Frecuencia y el Gasto.

df <- df %>%

df_pca <- df %>%

Para decidir el nmero de componentes que utilizar no existe un aregla escrita, los estadistas suelen fijarse en lo que se conoce como la regla del codo que dice que en algun punto del grfico la linea que hemos representado tender a hacer un codo lo suficientemente grande para tenerlo en cuenta. De esta manera cojeremos las variables hasta ese codo. En este caso nos quedaramos con la primera y segunda componente principal y rechazaramos la tercera.

Ahora, con los resultados que nos ha devuelto la funcin prcomp(), debemos aplicar una transformacin lineal a cada una de las columnas de nuestro dataframe con la matriz de rotacin.

df_pca_norm <- data.frame(df_pca_norm) %>%
  mutate(
    PC1 = apply(pca$rotation[,1]*df_pca_norm, 1, sum),
    PC2 = apply(pca$rotation[,2]*df_pca_norm, 1, sum)
  )

Sobre el grfico de dispersin de las componentes PC1 y PC2 vamos a aplicar un algoritmo de aprendizaje automtico no supervisado llamado kmeans. Que nos va a identificar dentro de los datos tantos grupos como le indiquemos. Los grupos los crea en funcion de la similitud entre los individuos de un universo, en este caso nuestro dataset compuesto por PC1 y PC2.

centroides <- 3
df_clusters <- kmeans(df_pca_norm[,c('PC1','PC2')], centroides,nstart=100, iter.max=100) #Le pasamos por parmetro la misma matriz normalizada que hemos usado para PCA.

groups <- as.character(df_clusters$cluster)

ggplot() +
  geom_point(data = df_pca_norm, 
             mapping = aes(x = PC1, 
                                  y = PC2, 
                                  colour = groups)) +
  config.estilos

Vemos que sobre el grfico anterior ahora podemos distinguir grupos que podemos analizar y dar un sentido.

Pero como sabemos que hay 3 y no 10 grupos en nuestros datos? Para ello debemos ejecutar el algoritmo kmeans iterativamente y ver cual nos devuelve mejores resultados.

Para automatizar la ejecucin del algoritmo de forma repetida podemos utilizar la funcin kmeansruns() que nos realizar las pruebas de forma autnoma y nos devolver todos los resultados de cada una de las iteraciones con los distintos valores de K. Adicionalmente vamos a realizar una representacin de los resultados que devuelve cada valor de K y con ello elegiremos el mejor nmero de clusters para nuestros datos.

clusters_ks <- kmeansruns(df_pca_norm[,c('PC1','PC2')], krange = 2:10, criterion = 'asw')

df_clusters_ks <- as.data.frame(scale(clusters_ks$crit))

ggplot(df_clusters_ks, aes(x = 1:10, y=V1)) +
  geom_point() +
  geom_line() +
  labs(
    title = 'Resultados para diferentes valores de K',
    x = 'k',
    y = 'Resultado'
  ) +
  scale_x_continuous(breaks = 1:10, labels = 1:10) +
  config.estilos

Volviendo a aplicar la regla del codo podemos tomar la decisin del valor de k que maximiza el resultado del indicador. En este caso vemos que 3 clusters explican muy bien nuestros datos, aun que el valor ptimo es de 6 clusters. Este va a ser el valor de k que vamos a elegir.

centroides <- clusters_ks$bestk
df_clusters <- kmeans(df_pca_norm[,c('PC1','PC2')], centroides,nstart=100, iter.max=100) #Le pasamos por parmetro la misma matriz normalizada que hemos usado para PCA.

groups <- as.character(df_clusters$cluster)

ggplot() +
  geom_point(data = df_pca_norm, 
             mapping = aes(x = PC1, 
                                  y = PC2, 
                                  colour = groups)) +
  config.estilos
